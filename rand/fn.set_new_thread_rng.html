pub fn set_new_thread_rng(creator: Box<Fn() -> Box<Rng>>)

Control how thread-local generators are created for new threads.

The first time thread_rng() is called in each thread, this closure is used to create a new generator. If this closure has not been set, the built-in closure returning a new boxed OsRng is used.

Calling this affects all users of thread_rng, random and random_with in all new threads (as well as existing threads which haven't called thread_rng yet), including libraries. Users should ensure the generator used is secure enough for all users of thread_rng, including other libraries.

This closure should not panic; doing so would kill whichever thread is calling thread_rng at the time, poison its mutex, and cause all future threads initialising a thread_rng (or calling this function) to panic.
